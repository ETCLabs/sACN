Detailed Description

The sACN Source API.

Components that send sACN are referred to as sACN Sources. Use this API to act as an sACN Source.

const SacnSourceUniverseConfig *  config,
bool  dirty_now 
)

Add a universe to an sACN source.

Adds a universe to a source. If dirty_now is true, the source will start sending values on the next call to sacn_source_process_sources(). If dirty_now is false, the applicaton must call sacn_source_set_dirty() to mark it ready for processing.

If the source is not marked as unicast_only, the source will add the universe to its sACN Universe Discovery packets.

Parameters
[in]handleHandle to the source to which to add a universe.
[in]configConfiguration parameters for the universe to be added.
[in]dirty_nowWhether or not to immediately mark the universe as dirty.
Returns
kEtcPalErrOk: Universe successfully added.
kEtcPalErrInvalid: Invalid parameter provided.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrExists: Universe given was already added to this source.
kEtcPalErrNotFound: Handle does not correspond to a valid source.
kEtcPalErrNoMem: No room to allocate additional universe.
kEtcPalErrSys: An internal library or system call error occurred.

Create a new sACN source to send sACN data.

This creates the instance of the source, but no data is sent until sacn_source_add_universe() is called.

Note that a source is considered as successfully created if it is able to successfully use any of the network interfaces listed in the passed in configuration. This will only return kEtcPalErrNoNetints if none of the interfaces work.

Parameters
[in]configConfiguration parameters for the sACN source to be created.
[out]handleFilled in on success with a handle to the sACN source.
[out]good_interfacesOptional. If non-NULL, good_interfaces is filled in with the list of network interfaces that were succesfully used.
Returns
kEtcPalErrOk: Source successfully created.
kEtcPalErrNoNetints: None of the network interfaces provided were usable by the library.
kEtcPalErrInvalid: Invalid parameter provided.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrNoMem: No room to allocate an additional source.
kEtcPalErrNotFound: A network interface ID given was not found on the system.
kEtcPalErrSys: An internal library or system call error occurred.

◆ sacn_source_destroy()

void sacn_source_destroy ( sacn_source_t  handle)

Destroy an sACN source instance.

Stops sending all universes for this source. The destruction is queued, and actually occurs on a call to sacn_source_process_sources() after an additional three packets have been sent with the "Stream_Terminated" option set. The source will also stop transmitting sACN universe discovery packets.

Even though the destruction is queued, after this call the library will no longer use the priority_buffer or values_buffer you passed in on your call to sacn_source_add_universe().

Parameters
[in]handleHandle to the source to destroy.

◆ sacn_source_process_sources()

size_t sacn_source_process_sources ( void  )

Process created sources and do the actual sending of sACN data on all universes.

Note: Unless you created the source with manually_process_source set to true, this will be automatically called by an internal thread of the module. Otherwise, this must be called at the maximum rate at which the application will send sACN.

Sends data for universes which have been marked dirty, and sends keep-alive data for universes which are haven't changed. Also destroys sources & universes that have been marked for termination after sending the required three terminated packets.

Returns
Current number of sources tracked by the library. This can be useful on shutdown to track when destroyed sources have finished sending the terminated packets and actually been destroyed.

◆ sacn_source_reset_networking()

etcpal_error_t sacn_source_reset_networking ( sacn_source_t  handle,
const SacnMcastNetintId *  netints,
size_t  num_netints,
SacnNetworkChangeResult *  good_interfaces 
)

Resets the underlying network sockets for the sACN source.

This is typically used when the application detects that the list of networking interfaces has changed.

After this call completes successfully, the all universes on a source are considered new & dirty by the per-address priority logic in sacn_source_process_sources.

If this call fails, the caller must call sacn_source_destroy(), because the source may be in an invalid state.

Note that the networking reset is considered successful if it is able to successfully use any of the network interfaces passed in. This will only return kEtcPalErrNoNetints if none of the interfaces work.

Parameters
[in]handleHandle to the source for which to reset the networking.
[in]netintsOptional array of network interfaces on which to send to the specified universe. If NULL, all available network interfaces will be used.
[in]num_netintsNumber of elements in the netints array.
[out]good_interfacesOptional. If non-NULL, good_interfaces is filled in with the list of network interfaces that were succesfully used.
Returns
kEtcPalErrOk: Source changed successfully.
kEtcPalErrNoNetints: None of the network interfaces provided were usable by the library.
kEtcPalErrInvalid: Invalid parameter provided.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrNotFound: Handle does not correspond to a valid receiver.
kEtcPalErrSys: An internal library or system call error occurred.
