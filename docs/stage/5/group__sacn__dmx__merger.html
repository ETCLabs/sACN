Detailed Description

The sACN DMX Merger API.

Configuration options for the sACN DMX Merger module.

This API provides a software merger for buffers containing DMX512-A start code 0 packets. It also uses buffers containing DMX512-A start code 0xdd packets to support per-address priority.

While this API is used to easily merge the outputs from the sACN Receiver API, it can also be used to merge your own DMX sources together, even in combination with the sources received via sACN.

When asked to calculate the merge, the merger shall evaluate the current source buffers and update two result buffers:

  • 512 bytes for the merged data values (i.e. "winning level"). These are calculated by using a Highest-Level-Takes-Precedence(HTP) algorithm for all sources that share the highest per-address priority.
  • 512 source identifiers (i.e. "winning source") to indicate which source was considered the source of the merged data value, or that no source currently owns this address.

◆ sacn_dmx_merger_add_source()

etcpal_error_t sacn_dmx_merger_add_source ( sacn_dmx_merger_t  merger,
const EtcPalUuid *  source_cid,
source_id_t *  source_id 
)

Adds a new source to the merger.

Adds a new source to the merger, if the maximum number of sources hasn't been reached. The filled in source id is used for two purposes:

  • It is the handle for calls that need to access the source data.
  • It is the source identifer that is put into the slot_owners buffer that was passed in the DmxMergerUniverseConfig structure when creating the merger.
Parameters
[in]mergerThe handle to the merger.
[in]source_cidThe sACN CID of the source.
[out]source_idFilled in on success with the source id.
Returns
kEtcPalErrOk: Source added successfully.
kEtcPalErrInvalid: Invalid parameter provided.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrNoMem: No room to allocate memory for this source, or the max number of sources has been reached.
kEtcPalErrExists: the source at that cid was already added.
kEtcPalErrSys: An internal library or system call error occurred.

Gets a read-only view of the source data.

Looks up the source data and returns a pointer to the data or NULL if it doesn't exist. This pointer is owned by the library, and must not be modified by the application. The pointer will only be valid until the source or merger is removed.

Parameters
[in]mergerThe handle to the merger.
[in]sourceThe id of the source.
Returns
The pointer to the source data, or NULL if the source wasn't found.

Updates the source data and recalculate outputs.

The direct method to change source data. This causes the merger to recalculate the outputs. If you are processing sACN packets, you may prefer dmx_merger_update_source_from_sacn().

Parameters
[in]mergerThe handle to the merger.
[in]sourceThe id of the source to modify.
[in]new_valuesThe new DMX values to be copied in. This may be NULL if the source is only updating the priority or address_priorities.
[in]new_values_countThe length of new_values. May be 0 if the source is only updating the priority or address_priorities.
[in]priorityThe universe-level priority of the source.
[in]address_prioritiesThe per-address priority values to be copied in. This may be NULL if the source is not sending per-address priorities, or is only updating other parameters.
[in]address_priorities_countThe length of address_priorities. May be 0 if the source is not sending these priorities, or is only updating other parameters.
Returns
kEtcPalErrOk: Source updated and merge completed.
kEtcPalErrInvalid: Invalid parameter provided.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrNotFound: Handle does not correspond to a valid source or merger.
