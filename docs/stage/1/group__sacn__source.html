Detailed Description

The sACN Source API.

Components that send sACN are referred to as sACN Sources. Use this API to act as an sACN Source.

◆ sacn_process_sources()

size_t sacn_process_sources ( void  )

Process created sources and do the actual sending of sACN data.

Must be called at the maximum rate at which the application will send sACN. Sends data for start codes which have been marked dirty, and sends keep-alive data for start codes which are static. Also destroys sources that have been marked for termination after sending the required three terminated packets.

Returns
Current number of sources tracked by the library. This can be useful on shutdown to track when destroyed sources have finished sending the terminated packets and actually been destroyed.

◆ sacn_source_add_start_code()

etcpal_error_t sacn_source_add_start_code ( sacn_source_t  handle,
const SacnStartCodeConfig *  sc_config 
)

Add a start code to an sACN source.

Provides a pointer to a buffer for the slot data for this start code; note that this slot data doesn't include the start code itself. The library retains ownership of this buffer and it is invalidated on sacn_source_remove_start_code() or sacn_source_destroy(). No data is sent until the start code data is marked as dirty using sacn_source_set_dirty() or passed in the dirty_handles parameter to sacn_process_sources().

Parameters
[in]handleHandle to the source to which to add a start code.
[in]sc_configConfiguration parameters for the start code to be added.
Returns
kEtcPalErrOk: Start code successfully added.
kEtcPalErrInvalid: Invalid parameter provided.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrExists: Start code given was already added to this universe.
kEtcPalErrNotFound: Handle does not correspond to a valid source.
kEtcPalErrNoMem: No room to allocate additional start code.
kEtcPalErrSys: An internal library or system call error occurred.

◆ sacn_source_destroy()

etcpal_error_t sacn_source_destroy ( sacn_source_t  handle)

Destroy an sACN source instance.

Stops sending all start codes for this source. The destruction is queued, and actually occurs on a call to sacn_process_sources() after three packets have been sent with the "Stream_Terminated" option set. After calling this function, any buffers provided by calls to sacn_source_add_start_code() for this universe should be considered invalid.

Parameters
[in]handleHandle to the source to destroy.
Returns
kEtcPalErrOk: Source destroyed successfully.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrNotFound: Handle does not correspond to a valid source.
kEtcPalErrSys: An internal library or system call error occurred.

◆ sacn_source_remove_start_code()

etcpal_error_t sacn_source_remove_start_code ( sacn_source_t  handle,
uint8_t  start_code 
)

Remove a start code from a universe.

If this is not the only start code being sent for this source, simply stops sending data for this start code immediately. Otherwise, if this is the last start code remaining on the source, flags the source for termination; data will stop after three packets are sent from sacn_process_sources() with the "Stream_Terminated" option sent. Either way, the buffer provided for this start code by sacn_source_add_start_code() should be considered invalid after calling this function.

Parameters
[in]handleHandle to the source from which to remove the start code.
[in]start_codeStart code to remove.
Returns
kEtcPalErrOk: Start code removed successfully.
kEtcPalErrNotInit: Module not initialized.
kEtcPalErrNotFound: Handle does not correspond to a valid source, or the start code was not previously added to this source.
kEtcPalErrSys: An internal library or system call error occurred.

Send the data for a set of source/start code combinations immediately.

This behavior is rarely necessary, but provided for unanticipated use cases and debug use. Use this function to send a sACN packet for a universe immediately (i.e. in between calls to sacn_process_sources()).

Note that calling this function could result in behavior which violates E1.31-2016 § 6.6.
